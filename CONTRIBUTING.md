# Contributing to GroundUp

Thank you for considering contributing to GroundUp! This document provides guidelines for contributing to the project.

## How to Contribute

### 1. Documentation Improvements

**What we're looking for:**
- Corrections to existing documentation
- Additional usage patterns and examples
- Clearer explanations of concepts
- Better organization of information

**Guidelines:**
- Keep content grounded in technical reality
- Avoid mystical or anthropomorphic language
- Cite sources when possible
- Use clear, accessible language

### 2. New Usage Patterns

**What we're looking for:**
- Novel approaches to working with AI systems
- Documented workflows with examples
- Pattern identification across different use cases
- Failure modes and how to avoid them

**Format:**
```markdown
## Pattern Name

**Context:** When to use this pattern
**Approach:** Step-by-step instructions
**Example:** Concrete example with prompts
**Pitfalls:** Common mistakes to avoid
**Variations:** Alternative approaches
```

### 3. Tool Comparisons

**What we're looking for:**
- Objective comparisons of AI systems
- Interface difference documentation
- Strength/weakness analysis
- Real-world testing results

**Guidelines:**
- Be objective and evidence-based
- Test claims before documenting
- Update comparisons as tools evolve
- Avoid marketing language

### 4. Examples and Templates

**What we're looking for:**
- Reusable prompt templates
- Workflow examples
- Integration patterns
- Edge case handling

**Requirements:**
- Must be tested and verified
- Should include expected outcomes
- Document failure scenarios
- Explain reasoning behind choices

## Submission Process

### For Small Changes
1. Fork the repository
2. Create a branch (`git checkout -b fix/typo-in-docs`)
3. Make your changes
4. Submit a pull request

### For Large Changes
1. Open an issue first to discuss the change
2. Wait for feedback before starting work
3. Follow the small changes process once approved

## Style Guidelines

### Markdown
- Use standard GitHub-flavored Markdown
- Include a table of contents for long documents
- Use code blocks with language tags
- Add alt text to images

### Tone
- Professional but accessible
- Avoid jargon where possible
- Define terms on first use
- Be concise

### Structure
- Use clear headings
- Break up long paragraphs
- Use lists for multiple points
- Include examples

## What NOT to Contribute

Please avoid:
- ❌ Claims about AI consciousness or sentience
- ❌ Mystical or spiritual interpretations
- ❌ Anthropomorphizing AI systems
- ❌ Unverifiable assertions
- ❌ Marketing or promotional content
- ❌ Personal attacks or inflammatory content

## Code of Conduct

All contributors must follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## Questions?

Open an issue with the tag `question` and we'll help you out.
